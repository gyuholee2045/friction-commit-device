# DWELL — 미디어 소스 제작 체크리스트

각 섹션 `미디어:` 줄을 실제 작업 단위로 분해. 무엇 / 어떻게 / 소스 / 길이·포맷 / 상태.

## 공통 캡처 셋업 (먼저 1회)
- **웹 소스 위치:** `localhost:3000/`(index.html = C 단독), `localhost:3000/compare.html`(A/B/C). `.claude/launch.json`의 `web-prototype` 기동.
- **권장 툴 (Windows):** OBS Studio = 고화질 MP4 화면녹화 / ShareX = 영역 GIF·MP4 + 캡처 / ffmpeg = 변환·무한루프 / VS Code Wokwi 확장 = 보드 시뮬 녹화.
- **중요 — GIF 대신 MP4 우선:** 유체/글래스 그라데이션은 GIF(256색)에서 띠(banding)가 생김. 물·3D·확산 장면은 전부 MP4(H.264), GIF는 단순 타이포/정적 장면에만.
- **공통 사양:** 2배 해상도로 녹화 후 다운스케일(최소 1920×1080), 60fps(유체 부드럽게), 무음. 루프는 ffmpeg로 앞뒤 이어붙여 끊김 제거.
- **비율:** Framer 레이아웃 확정 전 기본값 — 풀블리드 16:9, 섹션 인셋 16:9 또는 1:1. 확정 시 재크롭.

## 섹션별 자산

### M0 · Hero — 풀스크린 유체 루프
- **무엇:** index.html 유체 링이 잔잔히 일렁이는 idle 배경 루프(확정 동작 없음).
- **어떻게:** OBS 풀스크린 녹화 → ffmpeg 4~6초 심리스 루프.
- **소스:** `localhost:3000/` (누르지 않은 상태).
- **길이·포맷:** 4~6초 루프, MP4, 16:9 풀블리드(+ 모바일 1:1/9:16 별도 크롭).
- **상태:** 캡처 가능.

### M1 · The Tension — 무마찰 대비 배경
- **무엇:** (택1) 미니멀 키네틱 타이포 / 또는 원클릭에 즉시 지나가는 UI들이 흐려지며 스쳐가는 배경.
- **어떻게:** 타이포는 Framer 텍스트 애니메이션으로 대체(영상 0). 배경형은 compare.html A(즉시 전송) 짧게 녹화.
- **소스:** Framer 네이티브 / 또는 `compare.html` 토글 A.
- **길이·포맷:** 3~5초 루프 MP4, 또는 영상 없이 Framer 텍스트.
- **상태:** 캡처 가능. 결정 필요: 타이포 vs 배경 클립.

### M2 · Friction as Feature — 단일 메타포 클로즈업
- **무엇:** 유체 표면에 닿아 잠시 버티는 클로즈업(정지 + 미세 모션).
- **어떻게:** index.html 누름 시작 초반 0.5초를 크게 크롭해 미세 모션 루프.
- **소스:** `localhost:3000/` 누름 시작 구간 크롭.
- **길이·포맷:** 2~4초 루프 MP4, 1:1 또는 16:9 클로즈업.
- **상태/주의:** 캡처 가능. "손가락" 접촉은 웹엔 없음(커서/누름). 실제 손가락 샷은 실물 보드 단계로 보류 → 커서-누름 클로즈업으로 대체.

### M3 · The Approval Gate — 시나리오 셋업
- **무엇:** 340명에게 보내기 직전 장면 + 340 / 외부 12 / 임원 3 칩 스택.
- **어떻게:** compare.html 상단 정적 캡처(PNG), 또는 칩이 쌓이는 2~3초 모션.
- **소스:** `compare.html` 상단.
- **길이·포맷:** 정적 PNG(2x) 또는 2~3초 MP4, 16:9.
- **상태:** 확인 필요 — compare.html에 340/12/3 칩 UI 실재 여부.

### M4 · Three Ways — A/B/C 비교
- **무엇:** 세 조건 차이 나란히. (택1) compare.html 라이브 임베드 / A·B·C 3분할 클립.
- **어떻게:** 임베드면 Framer iframe. 아니면 A/B/C 각 5~7초 개별 녹화 후 3분할 또는 3열 배치.
- **소스:** `compare.html` 토글 A→B→C.
- **길이·포맷:** 각 5~7초 MP4 3개(물 포함 → GIF 비권장). 임베드면 라이브.
- **상태:** 캡처 가능. 결정 필요: 라이브 임베드 vs 3클립.

### M5 · The Held Moment — 핵심 인터랙션 + 적응형 데이터 (2개 1세트)
- **M5a 인터랙션 클립:** 홀드 → 단계 채움 → 확산. 소스 `localhost:3000/`. 5~8초 MP4, 16:9 또는 1:1. 상태: 캡처 가능.
- **M5b 적응형 escalation 증거:** 연타할수록 required_dwell가 base 800ms에서 상한(6000ms)으로 치솟는 시리얼 CSV. 표/클립.
  - 어떻게: Wokwi 연타 → 시리얼 모니터 CSV 캡처, 또는 CSV 값을 표/막대 그래픽으로 재구성.
  - 소스: Wokwi 시리얼 모니터.
  - 길이·포맷: 정적 표 PNG 또는 4~6초 스크롤 클립.
  - 상태: 부분 블로커 — Wokwi 시각/동작 확인 선행 필요. 그 전엔 코드 상수 기반 표로 임시 구성 가능.

### M6 · Two Embodiments — 하드웨어 증거 (3개 1세트)
- **M6a Wokwi 동작 클립:** 버튼 → LED 점진 점등 → COMMITTED 완전 점등 + 시리얼 CSV. VS Code Wokwi 확장에서 보드+시리얼 함께 녹화. 6~10초 MP4. 상태: 블로커 — Wokwi 시각 확인 미완.
- **M6b 테스트 통과 출력:** `pio test -e native` 20/20 PASS 터미널 화면. 스크린샷(PNG) 또는 carbon 정돈. 상태: 즉시 캡처 가능.
- **M6c 코어 코드 스니펫:** `friction_policy.cpp`의 `computeRequiredDwell` 또는 상태 전이 일부. 코드 이미지(carbon). 상태: 즉시 캡처 가능.
- **포맷:** 3분할 또는 순차 등장.

### M7 · The Making — 이터레이션 타임라인
- **무엇:** 2D → 3D 크롬 → 실리콘 → 표면 자기교차 버그 → 블롭 → 링, 각 한 줄 캡션 스트립.
- **어떻게:** 단계별 스틸/짧은 GIF를 가로 타임라인으로.
- **소스:** 과거 버전 캡처본.
- **길이·포맷:** 스틸 6컷 PNG 또는 각 2~3초 미니 클립 스트립.
- **상태:** 블로커/확인 필요 — 과거 이터레이션 캡처본 보유 여부 먼저 확인. 없으면 (a) 보유분만 (b) 일부 재현 녹화 (c) 섹션 축소. 가장 먼저 확인.

### M8 · Reflection / Scope — 차분한 배경
- **무엇:** 유체가 거의 멈춘 듯 미세하게 일렁이는 저강도 배경 + 정적 타이포.
- **어떻게:** M0 루프를 느리게/저대비로 변형하거나 idle 구간 재사용.
- **소스:** M0 재활용.
- **길이·포맷:** 6~8초 저속 루프 MP4, 16:9.
- **상태:** 캡처 가능(M0 파생).

### M9 · Outro — 마지막 확산 후 정지
- **무엇:** 유체가 한 번 잔잔히 확산 → 멈춤 + 링크 버튼(Live demo / Code).
- **어떻게:** index.html 확정(확산) 1회 녹화 후 마지막 프레임 정지. 버튼은 Framer 네이티브.
- **소스:** `localhost:3000/` 확정 동작.
- **길이·포맷:** 3~5초 MP4(루프 아님, 끝에서 정지) + Framer 버튼.
- **상태:** 캡처 가능.

## 작업 순서 (의존성 기준)
1. 먼저 확인할 블로커 2개: (a) M7 과거 이터레이션 캡처 보유 여부, (b) M3 칩 스택 UI 실재 여부.
2. 즉시 가능(웹): M0 → M8·M9(파생) → M2 → M4 → M5a. (웹 한 번 켠 김에 일괄 녹화.)
3. 즉시 가능(터미널/코드): M6b, M6c.
4. Wokwi 선행 필요: M6a, M5b. → Wokwi 시각 확인 후 한 번에 녹화.
5. 소재 정리: M7(보유분 기준), M1(타이포 vs 배경 결정).

## 정리가 필요한 결정 3가지
- M1: 키네틱 타이포(영상 없음) vs 무마찰 배경 클립
- M4: compare.html 라이브 임베드 vs A/B/C 3클립
- M5b/M6a: Wokwi 동작 확인을 지금 끝내고 캡처할지, 실물 보드까지 미룰지
